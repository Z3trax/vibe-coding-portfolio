# News Parser - Парсер новостей

Программа для автоматического сбора новостей с веб-сайтов.

## Описание

Скрипт `parser.py` отправляет HTTP-запрос к новостному сайту (Hacker News), то есть решает задачу парсинга со следующим функционалом:

- ✅ **GET-запрос с User-Agent** — имитирует браузер для предотвращения блокировки
- ✅ **BeautifulSoup для парсинга** — извлекает структурированные данные из HTML
- ✅ **Извлечение данных**:
  - Заголовок новости (текст)
  - Ссылка на статью (href)
  - Дата публикации (если доступна)
- ✅ **Сохранение результатов**:
  - `news.csv` — таблица с колонками: title, link, date
  - `news.json` — структурированный JSON формат
- ✅ **Обработка ошибок** — программа не упадёт при сбоях сети или изменении селекторов
- ✅ **Статистика** — вывод количества собранных новостей
- ✅ **Комментарии на русском** — код полностью задокументирован

### 🆕 Новые возможности:

- ✅ **Аргументы командной строки** — указание URL, имён файлов и других параметров
- ✅ **Логирование** — модуль `logging` для вывода в консоль и файл
- ✅ **Пауза между запросами** — `time.sleep()` для предотвращения перегрузки сервера
- ✅ **Парсинг нескольких URL** — чтение списка из файла (urls.txt)
- ✅ **Гибкая конфигурация** — настройка через аргументы командной строки

## Использование

### Базовое использование (по умолчанию):
```bash
python parser.py
```

### Все доступные аргументы:
```bash
python parser.py --help
```

### Примеры использования:

#### 1️⃣ Парсинг конкретного сайта:
```bash
python parser.py --url https://example.com
```

#### 2️⃣ Задание имён выходных файлов:
```bash
python parser.py --csv my_news.csv --json my_news.json
```

#### 3️⃣ Парсинг нескольких страниц из файла:
```bash
python parser.py --url-file urls.txt --csv all_news.csv
```

Содержимое `urls.txt`:
```
https://news.ycombinator.com/
https://news.ycombinator.com/newest
https://news.ycombinator.com/best
```

#### 4️⃣ Изменение задержки между запросами (в секундах):
```bash
python parser.py --delay 5  # Пауза 5 сек между запросами
```

#### 5️⃣ Логирование в файл:
```bash
python parser.py --log parser.log
```

#### 6️⃣ Подробное логирование (DEBUG уровень):
```bash
python parser.py --log-level DEBUG --log debug.log
```

#### 7️⃣ Комбинированный пример:
```bash
python parser.py --url-file urls.txt --csv news_all.csv --json news_all.json --delay 3 --log app.log --log-level INFO
```

## Адаптация для других сайтов

Чтобы парсить другой новостной сайт:

1. **Измените `NEWS_URL`** на целевой сайт
2. **Найдите селекторы**:
   - Откройте сайт в браузере
   - Кликните F12 → откроется DevTools
   - Инспектируйте элемент новости (правый клик → Inspect)
   - Найдите CSS-класс или селектор
3. **Обновите селекторы в функции `extract_news()`**

### Примеры селекторов для популярных сайтов:

- **Hacker News**: `.titleline > a` (текущая конфигурация)
- **BBC News**: `.sc-53a2e5d6-3` или `.sc-4fedabbc-3` (может меняться)
- **Reddit**: `h3, [data-testid="post-title"]`

## Обработка ошибок

Программа обрабатывает:
- ❌ Ошибки подключения
- ❌ Timeout (превышение времени ожидания)
- ❌ HTTP ошибки (404, 503 и т.д.)
- ❌ Отсутствие элементов в HTML
- ❌ Ошибки кодирования символов

## Лучшие практики

- ⚠️ **Уважайте `robots.txt`** — проверьте, разрешен ли парсинг сайта
- ⚠️ **Не перегружайте сервер** — добавьте задержку между запросами при массовом сборе
- ⚠️ **Следуйте Terms of Service** — не все сайты разрешают автоматический парсинг
- ⚠️ **Обновляйте селекторы** — сайты часто меняют HTML структуру
