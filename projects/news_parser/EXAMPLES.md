# News Parser - Примеры и советы использования

## 📚 Документация по новым возможностям

Программа была обновлена со следующими улучшениями:

### 1. ✅ Аргументы командной строки (argparse)

Все параметры теперь можно настраивать через аргументы командной строки.

**Основные аргументы:**

```bash
--url URL                  # URL сайта для парсинга (по умолчанию: https://news.ycombinator.com/)
--url-file FILE           # Файл со списком URL (игнорирует --url)
--csv FILE                # Имя CSV файла (по умолчанию: news.csv)
--json FILE               # Имя JSON файла (по умолчанию: news.json)
--delay SECONDS           # Пауза между запросами (по умолчанию: 2 сек)
--log FILE                # Файл для логирования (опционально)
--log-level LEVEL         # Уровень логирования: DEBUG/INFO/WARNING/ERROR (по умолчанию: INFO)
--help                    # Показать справку
```

### 2. 🌍 Логирование (logging модуль)

Вместо `print()` используется `logging`:
- Логирование в **консоль** всегда
- Опционально логирование в **файл** (--log параметр)
- **3 уровня детализации**: INFO, DEBUG, WARNING, ERROR

**Особенности:**
- Каждое сообщение имеет **timestamp** и **уровень**
- Форматированный вывод для лучшей читаемости
- Асинхронное логирование в файл (не замедляет программу)

### 3. ⏸ Пауза между запросами (time.sleep)

Автоматическая задержка перед каждым HTTP-запросом:

```bash
python parser.py --delay 2  # 2 секунды между запросами (по умолчанию)
python parser.py --delay 5  # 5 секунд между запросами (для медленных сайтов)
python parser.py --delay 0.5  # 0.5 секунд (быстрый парсинг)
```

**Зачем это нужно:**
- ✅ Предотвращает перегрузку серверов
- ✅ Избегает блокировки IP-адреса
- ✅ Вежливо относимся к хостажам сайтов

### 4. 📂 Чтение списка URL из файла

**Файл urls.txt:**
```
https://news.ycombinator.com/
# Комментарии начинаются с #
https://news.ycombinator.com/newest
https://news.ycombinator.com/best
```

**Использование:**
```bash
python parser.py --url-file urls.txt --csv all_news.csv --json all_news.json
```

**Особенности:**
- По одному URL на строку
- Пустые строки игнорируются
- Комментарии (начинаются с #) игнорируются
- Результаты объединяются в одном файле (режим append)

## 🚀 Практические примеры

### Пример 1: Базовый парсинг с логированием

```bash
python parser.py --log app.log
```

Результат:
- `news.csv` — новости в CSV
- `news.json` — новости в JSON
- `app.log` — логи выполнения

### Пример 2: Парсинг нескольких страниц

```bash
python parser.py --url-file urls.txt --csv hacker_news.csv --delay 3
```

**urls.txt:**
```
https://news.ycombinator.com/
https://news.ycombinator.com/newest
```

### Пример 3: Детальное логирование для отладки

```bash
python parser.py --url https://example.com --log-level DEBUG --log debug.log
```

Логи покажут:
- 🔍 Каждый загруженный URL
- 📊 Количество найденных элементов
- 🔗 Детали каждой извлеченной новости
- ❌ Все ошибки и предупреждения

### Пример 4: Работа с пользовательскими URL и файлами

```bash
python parser.py \
  --url-file my_sites.txt \
  --csv results/news.csv \
  --json results/news.json \
  --delay 2.5 \
  --log results/parser.log \
  --log-level INFO
```

### Пример 5: Быстрый парсинг одной страницы

```bash
python parser.py --url https://news.ycombinator.com/new --delay 0.5 --csv quick.csv
```

## 🔧 Обработка ошибок

Программа корректно обрабатывает:

```
❌ ConnectionError   → "Не удалось подключиться к сайту"
❌ Timeout          → "Превышено время ожидания (10 сек)"
❌ HTTPError 404    → "Ошибка HTTP 404"
❌ Missing selectors → "Новости не найдены"
❌ Encoding errors  → "Ошибка кодирования символов"
```

Программа **не падает** при ошибке, а продолжает работу и записывает ошибку в логи.

## 💡 Советы и лучшие практики

1. **Для надежности:** Используйте `--delay 2-3` для новостных сайтов
2. **Для скорости:** Используйте `--delay 0.5-1` для маленьких тестов
3. **Для отладки:** Используйте `--log-level DEBUG` и `--log debug.log`
4. **Для массового парсинга:** Создайте файл `urls.txt` с нужными URL
5. **Для мониторинга:** Используйте `--log app.log` для сохранения истории

## 📝 Создание файла с URL

**Пример my_urls.txt:**
```
# Технологические новости
https://news.ycombinator.com/
https://news.ycombinator.com/newest

# Другие источники (раскомментируйте для использования)
# https://example-news.com/tech
# https://another-site.com/headlines
```

**Использование:**
```bash
python parser.py --url-file my_urls.txt --csv tech_news.csv
```

## 🎯 Реальные сценарии использования

### Сценарий 1: Ежедневный мониторинг

```bash
#!/bin/bash
DATE=$(date +%Y-%m-%d)
python parser.py \
  --url https://news.ycombinator.com/ \
  --csv "news_$DATE.csv" \
  --log "logs/parser_$DATE.log"
```

### Сценарий 2: Мультиисточниковый парсинг

```bash
python parser.py \
  --url-file sources.txt \
  --csv combined_news.csv \
  --json combined_news.json \
  --delay 2 \
  --log combined.log
```

### Сценарий 3: Отладка селекторов

```bash
python parser.py \
  --url https://test-site.com \
  --log-level DEBUG \
  --log debug.log \
  --delay 1
```

Потом проанализируйте `debug.log`, чтобы увидеть, какие элементы были найдены.
